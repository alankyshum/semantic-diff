<objective>
Add word-level inline diff highlighting to modified lines, so users can see exactly which characters changed within a line, not just that the line was modified.

Purpose: This is the key readability feature (DIFF-03) that distinguishes a good diff viewer from a basic one. Without it, users must manually scan modified lines to find what actually changed.
Output: Modified line pairs show word-level highlighting with changed portions visually emphasized (e.g., bold + brighter background), while unchanged portions retain normal syntax highlighting.
</objective>

  <name>Task 1: Compute word-level inline diffs and render with emphasis</name>
  <files>src/diff/mod.rs, src/diff/parser.rs, src/ui/diff_view.rs</files>
  <action>
1. **src/diff/mod.rs** -- Add inline diff types:
   - `DiffSegment { tag: SegmentTag, text: String }` where `SegmentTag` is enum `{ Equal, Changed }`
   - `InlineDiff { old_segments: Vec<DiffSegment>, new_segments: Vec<DiffSegment> }` -- word-level diff between a removed/added line pair
   - Add `inline_diffs: Vec<InlineDiff>` field to `Hunk` struct (one per paired removed/added line group)
   - Or alternatively, store inline diff info per-line: add `inline_segments: Option<Vec<DiffSegment>>` to `DiffLine`

2. **src/diff/parser.rs** -- Compute inline diffs:
   - `pub fn compute_inline_diffs(hunk: &mut Hunk)` -- for each hunk, find consecutive removed+added line groups and pair them:
     - Walk through lines. When you see a sequence of Removed lines followed by a sequence of Added lines, pair them 1:1 (first removed with first added, etc). Extra lines in either group get no inline diff.
     - For each pair, use `similar::TextDiff::from_words(old_content, new_content)` to compute word-level diff
     - Map `similar::ChangeTag::Equal` -> `SegmentTag::Equal`, `Delete` -> `SegmentTag::Changed` (on old line), `Insert` -> `SegmentTag::Changed` (on new line)
     - Store segments in each DiffLine's `inline_segments` field
   - **Performance guard**: If either line in a pair is longer than 500 characters, skip inline diff for that pair (set inline_segments to None). This avoids O(ND) blowup on minified files.
   - **Whitespace handling**: Use `from_words` (not `from_chars`) so whitespace-only changes produce minimal diff noise
   - Call `compute_inline_diffs()` for each hunk during parsing in `parse()`, after building the Hunk

3. **src/ui/diff_view.rs** -- Render inline diff emphasis:
   - When rendering a DiffLine that has `inline_segments = Some(segments)`:
     - For each segment: if `tag == Equal`, render with normal syntax-highlighted style (from highlight cache). If `tag == Changed`, render with an emphasized style: **bold** text + brighter background (e.g., for added lines: `Color::Rgb(0, 80, 0)` instead of `Color::Rgb(0, 40, 0)`; for removed lines: `Color::Rgb(80, 0, 0)` instead of `Color::Rgb(40, 0, 0)`).
     - The emphasis overlays on top of the line's base +/- coloring, making changed words "pop" while unchanged words blend into the normal diff coloring.
   - When rendering a DiffLine with `inline_segments = None`: render as before (whole-line coloring, no word emphasis)
   - Implementation approach: instead of using the highlight cache spans directly, interleave them with the inline diff segments. For each segment, find which highlight cache spans overlap it and compose the styles (syntax color + diff emphasis background).
     - Simpler alternative if interleaving is complex: for lines WITH inline diff, skip syntax highlighting and just use the diff emphasis colors. This is acceptable for v1 -- syntax highlighting on non-inline-diff lines, emphasis-only on inline-diff lines.
  </action>
