2. **src/diff/mod.rs** -- Define and re-export the core data types:
   - `DiffData { files: Vec<DiffFile>, binary_files: Vec<String> }` -- top-level parsed result
   - `DiffFile { source_file: String, target_file: String, is_rename: bool, hunks: Vec<Hunk>, added_count: usize, removed_count: usize }` -- one changed file
   - `Hunk { header: String, source_start: usize, target_start: usize, lines: Vec<DiffLine> }` -- one @@ section
   - `DiffLine { line_type: LineType, content: String }` where `LineType` is enum `{ Added, Removed, Context }`
   - All types should derive Debug, Clone

3. **src/diff/parser.rs** -- `pub fn parse(raw: &str) -> DiffData`:
   - First, scan raw text for lines matching `^Binary files .+ and .+ differ$` -- extract file paths into `binary_files` vec
   - Then parse with `unidiff::PatchSet::new()` / `patch.parse(raw)` (use `unwrap_or_default()` to handle parse failures gracefully)
   - For each `PatchedFile`: detect renames by checking if `source_file.trim_start_matches("a/") != target_file.trim_start_matches("b/")` AND neither is `/dev/null`
   - Map each PatchedFile to DiffFile, each unidiff Hunk to our Hunk, each Line to DiffLine
   - Use `patched_file.added()` and `patched_file.removed()` for counts
   - Handle the "No newline at end of file" marker -- if a line starts with `\ No newline`, skip it or treat as context

4. **src/main.rs** -- Entry point:
   - FIRST: install panic hook that calls `ratatui::restore()` before the original hook (ROB-01). Do NOT use `panic = "abort"` in Cargo.toml.
   - Init tracing to `/tmp/semantic-diff.log` with env filter `semantic_diff=debug`
   - Run `git diff HEAD -M` via `std::process::Command`, capture stdout
   - If stdout is empty (no changes), print "No changes detected" to stderr and exit 0
   - Parse diff with `diff::parser::parse()`
   - Init terminal with `ratatui::init()`
   - Create App with parsed DiffData, call `app.run(terminal)`
   - After run returns, call `ratatui::restore()`
   - Use `#[tokio::main]` even though Phase 1 is sync -- sets up for Phase 2

Note: The `unidiff` crate API -- check actual API surface. The crate's `PatchSet` has method `parse()` which takes `&str`. Each file is accessed via iteration. Each file has `source_file`, `target_file`, methods `added()`, `removed()`. Each hunk is iterable from the file. Lines have `value` field and `line_type` field. If the API differs from this description, adapt accordingly -- the data types in mod.rs are what matter.

1. **src/app.rs** -- The TEA Model and Update:
   - `Message` enum: `KeyPress(crossterm::event::KeyEvent)`, `Quit`
   - `NodeId` enum: `File(usize)`, `Hunk(usize, usize)` -- derive Debug, Clone, Hash, Eq, PartialEq
   - `UiState` struct: `selected_index: usize`, `scroll_offset: u16`, `collapsed: HashSet<NodeId>`
   - `VisibleItem` enum: `FileHeader { file_idx }`, `HunkHeader { file_idx, hunk_idx }`, `DiffLine { file_idx, hunk_idx, line_idx }`
   - `App` struct: `diff_data: DiffData`, `ui_state: UiState`, `should_quit: bool`
   - `App::new(diff_data: DiffData) -> Self` -- initialize with all files expanded, selected_index 0
   - `App::run(&mut self, mut terminal: ratatui::DefaultTerminal) -> anyhow::Result<()>` -- main loop: draw, poll events (100ms timeout), handle key events
   - `App::update(&mut self, msg: Message)` -- dispatch to handle_key
   - `App::handle_key(&mut self, key: KeyEvent)` -- for now just handle `q`/`Esc` -> set should_quit = true
   - `App::visible_items(&self) -> Vec<VisibleItem>` -- flatten diff data respecting collapsed state (iterate files, if file not collapsed iterate hunks, if hunk not collapsed iterate lines)
   - `App::view(&self, frame: &mut ratatui::Frame)` -- delegate to `ui::draw(self, frame)`

2. **src/event.rs** -- Simple synchronous event polling for Phase 1:
   - This module is a placeholder for Phase 2's async event router
   - For now, the event loop lives directly in `App::run()` using `crossterm::event::poll()` and `crossterm::event::read()`
   - Export nothing yet -- the module exists so the file structure is ready

3. **src/ui/mod.rs** -- Basic rendering:
   - `pub fn draw(app: &App, frame: &mut ratatui::Frame)` -- renders the diff view
   - Layout: full terminal area, no sidebar yet
   - For each visible item, render a Line:
     - `FileHeader`: bold white text showing filename (show "renamed: old -> new" for renames, "[Binary] filename" for binary files from diff_data.binary_files). Show +N/-M counts in green/red.
     - `HunkHeader`: dim cyan text showing the @@ header
     - `DiffLine`: prefix with +/-/space, color green for added, red for removed, default for context. Show line content.
   - Use a `ratatui::widgets::Paragraph` built from all visible Lines, with `.scroll((app.ui_state.scroll_offset, 0))`
   - Show a summary line at the bottom: "N files changed, +A/-R lines" using diff_data totals
   - Highlight the selected_index line with a distinct background (e.g., dark blue or gray background)

Wire everything together so `cargo run` in a git repo with changes shows a basic diff view and `q` quits cleanly.
