  <name>Task 1: Harden signal.rs -- secure PID directory, atomic write, ownership validation</name>
  <files>src/signal.rs</files>
  <behavior>
    - Test 1: pid_dir() returns path under XDG_RUNTIME_DIR when env var is set
    - Test 2: pid_dir() falls back to $HOME/.local/state/semantic-diff when XDG_RUNTIME_DIR not set
    - Test 3: write_pid_file creates file with correct PID content
    - Test 4: read_pid returns None for non-existent file
    - Test 5: validate_pid_ownership returns false for PID 0 or obviously invalid PIDs
    - Test 6: validate_pid_ownership returns false when /proc/{pid}/comm doesn't match (Linux) -- cfg(target_os = "linux") only
    - Test 7: Atomic write uses temp file + rename pattern (verify by checking file exists after write)
  </behavior>
  <action>
Rewrite src/signal.rs to fix FINDING-06, 07, 08, 09:

1. Replace the hardcoded `/tmp/semantic-diff.pid` constant with a function `pid_dir() -> PathBuf` that returns:
   - `$XDG_RUNTIME_DIR/semantic-diff/` if the env var is set (typically `/run/user/<uid>/`)
   - Falls back to `$HOME/.local/state/semantic-diff/` if XDG_RUNTIME_DIR is not set
   - Create the directory with mode 0o700 if it doesn't exist (`DirBuilder::new().mode(0o700).create()` on Unix)

2. Compute PID file path as `pid_dir().join("semantic-diff.pid")` -- includes the app name so it's in its own subdirectory.

3. Rewrite `write_pid_file()` to use atomic write:
   - Write PID to a temp file in the same directory: `pid_dir().join(".semantic-diff.pid.tmp")`
   - Use `OpenOptions::new().write(true).create_new(true).mode(0o600)` with O_NOFOLLOW semantics (create_new fails if file exists, preventing symlink follow)
   - Call `std::fs::rename(tmp_path, pid_path)` for atomic replacement
   - If temp file already exists, remove it first then retry

4. Rewrite `read_pid()` to add ownership validation:
   - Read PID from file as before
   - Add `validate_pid_ownership(pid: u32) -> bool`:
     - On Linux: read `/proc/{pid}/comm` and check it contains "semantic-diff" (or check `/proc/{pid}/cmdline`)
     - On macOS: use `Command::new("ps").args(["-p", &pid.to_string(), "-o", "comm="]).output()` and check output contains "semantic-diff"
     - Return false if the process doesn't exist or doesn't match
   - `read_pid()` should return `None` if validation fails

5. Keep the same public API signatures so main.rs callers don't need changes (except the return value of read_pid which now validates).

6. Use `#[cfg(unix)]` for the mode/permissions bits. Use `std::os::unix::fs::OpenOptionsExt` for `.mode()`.

7. Add `pub fn pid_file_path() -> PathBuf` as a public getter so main.rs log path can use the same base directory.
  </action>

  </verify>
