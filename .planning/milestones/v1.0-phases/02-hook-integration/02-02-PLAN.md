   - `pub active_filter: Option<String>` — the confirmed filter (set on Enter in search mode)

3. Modify `handle_key()` to branch on `input_mode`:

   **When `InputMode::Normal`:**
   - `/` → set `input_mode = InputMode::Search`, clear `search_query`. Return None.
   - `n` → if `active_filter` is Some, jump to next matching file header in visible_items. Return None.
   - `N` → if `active_filter` is Some, jump to previous matching file header. Return None.
   - `Escape` → if `active_filter` is Some, clear `active_filter` and reset selected_index. Return None. If no active_filter, quit as before.
   - All other keys → existing behavior unchanged.

   **When `InputMode::Search`:**
   - `Escape` → set `input_mode = InputMode::Normal`, clear `search_query`, clear `active_filter`. Return None.
   - `Enter` → set `input_mode = InputMode::Normal`, set `active_filter = if search_query.is_empty() { None } else { Some(search_query.clone()) }`, reset `selected_index` to 0. Return None.
   - `Backspace` → pop last char from `search_query`. Return None.
   - `Char(c)` → push `c` to `search_query`. Return None.
   - All other keys → ignore (return None).

4. Modify `visible_items()` to filter when `active_filter` is Some:
   - When `active_filter` is Some(ref pattern), only include files whose `target_file` path contains the pattern (case-insensitive using `.to_lowercase().contains(&pattern.to_lowercase())`).
   - Hunks and lines of matching files still appear normally (respecting collapse state).
   - When `active_filter` is None, behavior is unchanged.

5. For `n`/`N` navigation: find the next/previous `VisibleItem::FileHeader` in visible_items() whose file path matches the filter, and set selected_index to that position.

  <files>.claude/hooks/refresh-semantic-diff.sh, .claude/settings.local.json</files>
  <action>
1. Create `.claude/hooks/` directory if it does not exist.

2. Create `.claude/hooks/refresh-semantic-diff.sh`:
   ```bash
   #!/bin/bash
   # PostToolUse hook: refresh semantic-diff or launch it in a cmux split
   # Called by Claude Code after Edit/Write tool calls (async, non-blocking)
   PIDFILE="/tmp/semantic-diff.pid"

   if [ -f "$PIDFILE" ]; then
       PID=$(cat "$PIDFILE")
       # Verify the process is actually semantic-diff (macOS compatible)
       if ps -p "$PID" -o comm= 2>/dev/null | grep -q semantic-diff; then
           kill -USR1 "$PID" 2>/dev/null
           exit 0
       fi
       # Stale PID file — remove it
       rm -f "$PIDFILE"
   fi

   # semantic-diff not running — launch in cmux split if available
   if command -v cmux >/dev/null 2>&1; then
       # Create a right split and send the launch command
       NEW_SURFACE=$(cmux new-split right --json 2>/dev/null | jq -r '.surface // empty')
       if [ -n "$NEW_SURFACE" ]; then
           cmux send --surface "$NEW_SURFACE" "cd \"${CLAUDE_PROJECT_DIR:-.}\" && semantic-diff\n"
       else
           # Fallback: try without --json if cmux version doesn't support it
           cmux new-split right 2>/dev/null
           # Cannot target the new pane without surface ID — user may need to run semantic-diff manually
       fi
   fi

   exit 0
   ```

3. Make the script executable: `chmod +x .claude/hooks/refresh-semantic-diff.sh`

4. Create `.claude/settings.local.json`:
   ```json
   {
     "hooks": {
       "PostToolUse": [
         {
           "matcher": "Edit|Write",
           "hooks": [
             {
               "type": "command",
               "command": "$CLAUDE_PROJECT_DIR/.claude/hooks/refresh-semantic-diff.sh",
               "async": true,
               "timeout": 10
             }
           ]
         }
       ]
     }
   }
   ```

5. Note: The script uses `$CLAUDE_PROJECT_DIR` which Claude Code sets automatically. The `async: true` flag ensures the hook does not block Claude Code. The `timeout: 10` gives 10 seconds for cmux split creation if needed.
