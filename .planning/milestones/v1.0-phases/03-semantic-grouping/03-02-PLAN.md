1. In `src/app.rs`, add new types and state:
   - Add `pub use crate::ui::file_tree::TreeNodeId;` (or define TreeNodeId here -- see below).
   - Add `FocusedPanel` enum: `FileTree`, `DiffView` (default: DiffView to preserve existing UX).
   - Add to `App` struct:
     - `pub focused_panel: FocusedPanel` -- which panel has keyboard focus
     - `pub tree_state: tui_tree_widget::TreeState<TreeNodeId>` -- persistent tree state for tui-tree-widget
   - Init in `App::new()`: `focused_panel: FocusedPanel::DiffView`, `tree_state: TreeState::default()`
   - Add Tab key handling in `handle_key_normal()`:
     ```rust
     KeyCode::Tab => {
         self.focused_panel = match self.focused_panel {
             FocusedPanel::FileTree => FocusedPanel::DiffView,
             FocusedPanel::DiffView => FocusedPanel::FileTree,
         };
         None
     }
     ```
   - When `focused_panel == FocusedPanel::FileTree`, route j/k/Up/Down/Enter/Left/Right to tree navigation instead of diff navigation. Use `tree_state.key_up()`, `tree_state.key_down()`, `tree_state.toggle_selected()`, `tree_state.key_left()`, `tree_state.key_right()` methods from tui-tree-widget.
   - When a File node is selected in the tree (Enter on a leaf), find the file index in diff_data.files and scroll the diff view to that file's FileHeader position. Use `visible_items()` to find the matching FileHeader index and set `ui_state.selected_index`.
   - Keep existing diff view navigation when `focused_panel == FocusedPanel::DiffView`.

2. Create `src/ui/file_tree.rs`:
   - Define `TreeNodeId` enum:
     ```rust
     #[derive(Debug, Clone, Hash, Eq, PartialEq)]
     pub enum TreeNodeId {
         Group(usize),
         File(String),  // normalized file path (no b/ prefix)
     }
     ```
     Implement `Display` for TreeNodeId (required by tui-tree-widget).
   - `pub fn build_tree_items(app: &App) -> Vec<TreeItem<'_, TreeNodeId>>`:
     - If `app.semantic_groups` is `Some(groups)`: build grouped tree. Each group is a `TreeItem::new(TreeNodeId::Group(i), styled_header, children)` where the header shows `"label (+N -M, K files)"` with summed stats from the group's files. Children are `TreeItem::new_leaf(TreeNodeId::File(path), styled_line)` for each file in the group. If any diff files are not in any group, add an "Other" catch-all group at the end.
     - If `app.semantic_groups` is `None`: build flat list. Each file is a `TreeItem::new_leaf(TreeNodeId::File(path), "path +N -M")`.
     - For file path matching: strip `b/` prefix from `DiffFile.target_file` when comparing to group file paths (which have no prefix). Use `ends_with()` for flexible matching.
     - Use `ratatui::text::Line` with styled `Span`s for tree item text: group labels in bold cyan, file paths in default, stats in green(+)/red(-).
   - `pub fn render_tree(app: &App, frame: &mut Frame, area: Rect)`:
     - Build tree items via `build_tree_items(app)`
     - Create `Tree::new(&items)` with:
       - `.block(Block::bordered().title("Files"))` -- use cyan border when FileTree is focused, default otherwise
       - `.highlight_style(Style::default().fg(Color::Black).bg(Color::Cyan).add_modifier(Modifier::BOLD))`
       - `.highlight_symbol(">> ")`
       - `.node_closed_symbol("> ")`
       - `.node_open_symbol("v ")`
       - `.node_no_children_symbol("  ")`
     - `frame.render_stateful_widget(tree, area, &mut app.tree_state)` -- NOTE: this requires mutable access. Since `app` is passed as `&App`, you need to wrap `tree_state` in `std::cell::RefCell` in the App struct, or change the render signature to take `&mut App`. The `RefCell` approach is simpler:
       - In App: `pub tree_state: std::cell::RefCell<tui_tree_widget::TreeState<TreeNodeId>>`
       - In render: `frame.render_stateful_widget(tree, area, &mut app.tree_state.borrow_mut())`
     - Show a small loading indicator in the block title when `app.grouping_status == GroupingStatus::Loading`: `"Files [grouping...]"`.

3. Update `src/ui/mod.rs`:
   - Add `pub mod file_tree;`
   - Change `draw()` to use a horizontal split layout:
     ```rust
     // Horizontal split: sidebar | diff view
     let horizontal = Layout::horizontal([
         Constraint::Max(40),   // sidebar capped at 40 cols
         Constraint::Min(60),   // diff view gets the rest
     ]).split(vertical[0]);

     // Render tree sidebar in left panel
     file_tree::render_tree(app, frame, horizontal[0]);

     // Render diff view in right panel (existing)
     diff_view::render_diff(app, frame, horizontal[1]);
     ```
   - Keep the bottom bar (search/summary) spanning full width as before.
   - Highlight the focused panel: when DiffView is focused, give diff_view area a visible border or use the existing style. When FileTree is focused, the tree block already gets a cyan border (handled in file_tree.rs).

4. Update `src/ui/summary.rs`:
   - Show grouping status in the summary bar. After the existing file count / +/- totals, append:
     - `GroupingStatus::Loading` -> ` | Grouping...`
     - `GroupingStatus::Done` -> ` | N groups` (where N = semantic_groups.len())
     - `GroupingStatus::Error(_)` -> ` | Ungrouped` (subtle, no alarm)
     - `GroupingStatus::Idle` -> nothing extra (claude not available or no groups requested)

  KEY UX DETAILS:
  - When transitioning from flat to grouped tree, try to preserve the selected file. If the currently selected TreeNodeId::File was selected, it should remain selected in the grouped tree since the identifier is the same path string.
  - The tree_state persists across rebuilds of tree items because tui-tree-widget matches by identifier. As long as TreeNodeId values are stable (they are -- file paths don't change), open/closed state is preserved.
  - Sidebar width: Max(40) ensures it doesn't take too much space. On narrow terminals (<80 cols), consider hiding the sidebar entirely or making it narrower. A simple approach: if terminal width < 80, use Constraint::Max(25).
