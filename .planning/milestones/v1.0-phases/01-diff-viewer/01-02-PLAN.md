<task type="auto">
  <name>Task 1: Implement syntax highlighting cache and keyboard navigation</name>
  <files>src/highlight.rs, src/app.rs</files>
  <action>
1. **src/highlight.rs** -- Syntax highlighting with cache:
   - `HighlightCache` struct holding a `HashMap<(usize, usize, usize), Vec<(ratatui::style::Style, String)>>` keyed by (file_idx, hunk_idx, line_idx)
   - `HighlightCache::new(diff_data: &DiffData) -> Self` -- pre-compute all highlights at construction time:
     - For each file, detect syntax from filename extension using `syntect::parsing::SyntaxSet::load_defaults_newlines()`
     - Use `syntect::highlighting::ThemeSet::load_defaults()` and pick `"base16-ocean.dark"` theme
     - For each line in each hunk, run `HighlightLines::highlight_line()` to get styled regions
     - Map syntect `Style` to ratatui `Style`: extract foreground RGB from `syntect_style.foreground` -> `ratatui::style::Color::Rgb(r, g, b)`
     - Store in cache HashMap
   - `HighlightCache::get(&self, file_idx, hunk_idx, line_idx) -> Option<&Vec<(Style, String)>>` -- lookup cached spans
   - Handle errors gracefully: if syntect fails for a line, store the raw text with default style

2. **src/app.rs** -- Add navigation and collapse/expand:
   - Add `highlight_cache: HighlightCache` field to App, construct in `App::new()`
   - **Navigation** (NAV-01): In `handle_key`:
     - `j`/`Down` -> move selected_index forward by 1 (clamp to 0..visible_items.len()-1)
     - `k`/`Up` -> move selected_index backward by 1
     - `g` -> jump to first item (index 0)
     - `G` -> jump to last item
     - `Ctrl+d` -> page down (move selected_index by half terminal height)
     - `Ctrl+u` -> page up
   - **Collapse/expand** (NAV-02, NAV-03): In `handle_key`:
     - `Enter` on a `FileHeader` -> toggle `NodeId::File(file_idx)` in collapsed set
     - `Enter` on a `HunkHeader` -> toggle `NodeId::Hunk(file_idx, hunk_idx)` in collapsed set
     - `Enter` on a `DiffLine` -> no-op
   - **Selection tracking**: After any collapse/expand, recompute visible_items. If selected_index >= new length, clamp to last item. Better: remember the NodeId of the selected item, find its new position, fall back to clamping if the item was collapsed away.
   - **Scroll tracking**: Compute scroll_offset from selected_index so the selected item is always visible in the viewport. If selected_index < scroll_offset, set scroll_offset = selected_index. If selected_index >= scroll_offset + viewport_height, set scroll_offset = selected_index - viewport_height + 1.
   - Make `highlight_cache` accessible to the view layer (pub field or getter method)
  </action>

<task type="auto">
  <name>Task 2: Build diff view and summary widgets with line numbers and statistics</name>
  <files>src/ui/mod.rs, src/ui/diff_view.rs, src/ui/summary.rs</files>

1. **src/ui/diff_view.rs** -- Dedicated diff rendering widget:
   - `pub fn render_diff(app: &App, frame: &mut Frame, area: Rect)` -- renders the diff view in the given area
   - For each visible item in the viewport (from scroll_offset to scroll_offset + area.height):
     - **FileHeader**: Render as bold line. Format: icon + filename (or "renamed: old -> new" for renames, "[Binary] name" for binary files). Right-align "+N -M" in green/red. Use a distinct background (e.g., dark gray) to visually separate files.
     - **HunkHeader**: Render @@ header line in dim cyan. Show collapse indicator (e.g., "v" expanded, ">" collapsed) before the @@ text.
     - **DiffLine**: Render with:
       - Line number gutter: show source line number for removed/context lines, target line number for added/context lines. Right-align in a 4-char-wide gutter. Use dim gray color.
       - +/-/space prefix in appropriate color
       - Syntax-highlighted content from `app.highlight_cache.get(file_idx, hunk_idx, line_idx)` -- compose each cached span into a `ratatui::text::Span` with the cached style, overlaid on a background color (dark green for added, dark red for removed, default for context)
       - If no cached highlight, fall back to plain text with just the +/- coloring
   - Selected item: apply a highlight background (e.g., `Color::Rgb(40, 40, 60)`) to the entire line
   - Collapse indicators: show "v"/">" arrow before file headers and hunk headers to indicate expanded/collapsed state

2. **src/ui/summary.rs** -- Summary bar widget:
   - `pub fn render_summary(app: &App, frame: &mut Frame, area: Rect)` -- renders summary in given area
   - Content: "{N} file(s) changed  +{A} -{R}" where N = file count + binary file count, A = total added lines, R = total removed lines
   - Style: bold text, green for +A, red for -R
   - If binary files exist, append " ({B} binary)" in yellow

3. **src/ui/mod.rs** -- Compose layout:
   - Update `draw()` to split the frame into two areas using `Layout::vertical`:
     - Main area (all remaining space) -> `diff_view::render_diff()`
     - Bottom bar (1 line) -> `summary::render_summary()`
   - Pass the correct areas to each sub-renderer
   - Remove old inline rendering code from Plan 01 (it gets replaced by diff_view)
