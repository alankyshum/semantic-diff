   - `cargo audit --json >> .planning/phases/04-red-team-audit/cargo-audit-output.txt 2>&1`

4. Note: cargo audit may return non-zero exit code if vulnerabilities are found -- this is expected and fine. Capture output regardless.

NO code changes. This task only runs tools and captures output.

<task type="auto">
  <name>Task 2: Bootstrap cargo deny and run all checks</name>
  <files>deny.toml, .planning/phases/04-red-team-audit/cargo-deny-output.txt</files>
  <action>
1. Install cargo-deny if not already installed: `cargo install cargo-deny`
2. Generate starter deny.toml config: `cargo deny init` (only if deny.toml does not already exist)
3. Run all cargo deny checks and capture output:
   - `cargo deny check 2>&1 | tee .planning/phases/04-red-team-audit/cargo-deny-output.txt`
4. If the full check fails, also run individual categories to get granular output:
   - `cargo deny check advisories 2>&1 >> .planning/phases/04-red-team-audit/cargo-deny-output.txt`
   - `cargo deny check licenses 2>&1 >> .planning/phases/04-red-team-audit/cargo-deny-output.txt`
   - `cargo deny check bans 2>&1 >> .planning/phases/04-red-team-audit/cargo-deny-output.txt`
   - `cargo deny check sources 2>&1 >> .planning/phases/04-red-team-audit/cargo-deny-output.txt`
5. Note: cargo deny may return non-zero exit code if issues are found -- this is expected. Capture all output.

NO code changes beyond creating deny.toml (config file, not source code).
  </action>
  <verify>
